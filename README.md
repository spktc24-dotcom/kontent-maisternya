# Контент-майстерня

Безкоштовний для користувачів генератор контенту й реклами для блогу: 8 гілок (тема/оффер,
шапка профілю, програма продукту, Reels, каруселі, Stories, рекламні креативи, лід-магніти)
з урахуванням маркетингу, алгоритму Instagram і запуску таргетованої реклами.

## Як це влаштовано (важливо прочитати)

Раніше сайт намагався ходити напряму в Anthropic API з браузера — це працює **тільки**
всередині артефактів Claude.ai. Якщо відкрити файл локально або захостити на GitHub Pages,
у браузера немає ключа доступу, тож генерація падає з помилкою.

Тому тепер все влаштовано інакше, і працює скрізь:

```
Браузер (сайт)  ──POST /api/generate──▶  Express-сервер (server.js)  ──▶  Anthropic API
Telegram (бот)  ─────────────────────▶  той самий сервер (bot.js)    ──▶  Anthropic API
```

Один Node.js-сервер (`server.js`) одночасно:
1. Роздає сайт (`public/index.html`),
2. Приймає запити генерації від сайту через `/api/generate`,
3. Запускає Telegram-бота, який ходить у ту саму логіку генерації.

Anthropic API-ключ зберігається **тільки на сервері** (у змінних середовища), ніколи
не потрапляє в браузер — це і безпечніше, і єдиний спосіб, щоб сайт працював поза Claude.ai.

**Наслідок:** GitHub Pages для цього проєкту не підходить (це просто статичний хостинг файлів
без сервера). Весь проєкт — сайт + API + бот — розгортається одним деплоєм на Render/Railway/Fly.io.
GitHub потрібен лише як сховище коду, з якого ці сервіси автоматично деплоять.

## 2. Деплой (сайт + API + бот одразу)

### Крок 1 — отримай Anthropic API-ключ
1. Зареєструйся на [console.anthropic.com](https://console.anthropic.com).
2. **Settings → API Keys → Create Key**.
3. Поповни баланс (навіть $5-10 вистачить надовго для особистого використання).

### Крок 2 — (опційно) створи Telegram-бота
1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot`, дай ім'я й юзернейм.
2. Збережи виданий токен. Якщо бот не потрібен — просто не задавай `BOT_TOKEN`,
   і запуститься тільки сайт.

### Крок 3 — деплой на Render (безкоштовний варіант)
1. Зайди на [render.com](https://render.com) → **New → Web Service**.
2. Підключи свій GitHub-репозиторій.
3. Build command: `npm install`, Start command: `npm start`.
4. У розділі **Environment** додай змінні:
   - `ANTHROPIC_API_KEY` = твій ключ
   - `BOT_TOKEN` = токен бота (якщо є)
5. Deploy. Через кілька хвилин Render видасть посилання виду
   `https://kontent-maisternya.onrender.com` — це і є твій сайт, і бот на цей момент
   вже теж запущений і відповідає в Telegram.

Аналогічно можна деплоїти на Railway чи Fly.io — принцип той самий: підключити репозиторій
і задати ті самі змінні середовища.

> ⚠️ Безкоштовні тарифи хостингів час від часу змінюють умови (наприклад, "засинання"
> сервісу без активності). Перевір актуальні умови на сайті провайдера перед деплоєм.

### Крок 4 — перевір локально (необов'язково, але корисно)
```bash
npm install
ANTHROPIC_API_KEY=твій_ключ BOT_TOKEN=твій_токен npm start
```
Відкрий `http://localhost:3000` — сайт має генерувати контент. Якщо задав `BOT_TOKEN`,
бот одразу відповідає в Telegram.

## Структура репозиторію

```
├── server.js            — точка входу: сайт + API + запуск бота
├── content.js            — спільні картки, системний промпт і виклик Anthropic API
├── bot.js                 — логіка Telegram-бота
├── public/
│   └── index.html          — сайт (фронтенд, ходить у /api/generate)
├── package.json
├── .env.example            — приклад змінних середовища
├── .gitignore
└── analiz-dodatkiv.md      — звіт з аналізом первинних додатків
```

## Про витрати

- Хостинг на Render/Railway — безкоштовний тариф вистачає для особистого проєкту.
- Anthropic API — постійного безкоштовного тарифу немає, є невеликий пробний кредит для
  нових акаунтів. Кожна генерація коштує частки цента. Якщо сайт/бот стане популярним,
  має сенс додати ліміт генерацій на людину на день — скажи, і я додам цю логіку.
